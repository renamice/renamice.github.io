Hack Assembler

Full code can be found at github.

It had been quite some time since I started off the Nand2Tetris Part I course on coursera. Because I wasn’t paying for it I wasn’t under the pressure of finishing the assignments but I still wanted to do it anyways but the last week’s assignment was kind of a daunting task for some reason.
They ask you to build an assembler for the hack assembly, which the computer that you build throughout the course utilizes, and they also provide a complete breakdown of the logic, design, and tips on how to approach it.

This assembler, however, isn’t an “actual assembler” if you consider assemblers to be able to generate executables. This one is focused on a very high level abstracted implementation. As such, you have to translate the hack assembly into binary and write that binary onto a file as ASCII characters. So, instead of an executable file you output a .hack file that contains 0s and 1s.

I have also wanted to learn rust for quite some time now and wanted to code this entire thing is it so it would help in both areas. I coded this thing throughout my saturday and it took me around 10 hours to finish. I have managed to make something that some what works.

They recommend that you first build a version that can work with code that have no symbols and this one was pretty easy to do. This part of the project was done in the first 4 hours. With this version working correctly for the files that it needs to work for I started building another version where it would support symbols.
For some reason or another I couldn’t figure this one out. My last version works for all the files that they provide for testing except one: Pong.asm.
I have no idea why this file alone doesn’t assemble correctly.

I could try to debug the mess that I have created but I don’t even know where to begin. It works for all the other files so I have no idea what’s broken in the first place.

For the start I started off with handling command line arguments in rust.\

fn argument_handling() -> (String, String) {
    let args: Vec<String> = env::args().collect();

    if args.len() == 1 {
        panic!("no arguments provided!")
    }

    let file_name: &String = &args[1];

    if file_name == "-h" {
        help();
    }
    let file = load_file(&file_name);

    let file: String = match file {
        Ok(code) => code,
        Err(error) => {
            panic!("error: {:?}", error)
        }
    };
    let output_file = file_name.replace("asm", "").to_owned() + &"hack";
    return (file, output_file);
}

This does multiple things:

  1. prints out no arguments provided! if nothing is provided.\
  2. calls the help function if -h is provided:
fn help() {

Looking back, I could have just done this in the host function itself, but it is what it is.

  1. if none of the above cases match, it attemps to load the file and returns it along with <file_name>.hack as output file name.

The main function then goes through this returned string which contains the file, cleans it up for any comments or whitespaces, breaks it down into a vector, and passes eachline to handle_instruction. The function sends the line to either of two other functions: a_instruction, and c_instruction.
Both of these make uses of global hashmaps generated with lazy_static.

fn a_instruction(instruction: &String) -> String {
    // in the form of "@<address>"
    let instruction = instruction.replace("@", "");
    let address = instruction.parse::<u32>().unwrap();
    let address = binary_converter(address);

    let left_space: usize = 15 - address.len();
    let mut into_binary: String = String::from("0");

    for _ in 0..left_space {
        into_binary.push('0');
    }
    into_binary.push_str(&address);
    return into_binary;
}

fn c_instruction(instruction: &String) -> String {
    // syntax: "<dest>=<comp>;<jump>"
    // binary: 111 a c1 c2 c3 c4 c5 c6 d1 d2 d3 j1 j2 j3

    let mut dest: String = String::from("null");
    let mut comp: String = String::from("0");
    let mut jump: String = String::from("null");
    if instruction.contains("=") && instruction.contains(";") {
        let components: Vec<&str> = instruction
            .split(&['=', ';'])
            .filter(|&r| r != "")
            .collect();
        dest = components[0].to_string();
        comp = components[1].to_string();
        jump = components[2].to_string();
    } else if instruction.contains("=") {
        let components: Vec<&str> = instruction.split(&['=']).filter(|&r| r != "").collect();
        dest = components[0].to_string();
        comp = components[1].to_string();
    } else if instruction.contains(";") {
        let components: Vec<&str> = instruction.split(&[';']).filter(|&r| r != "").collect();
        comp = components[0].to_string();
        jump = components[1].to_string();
    }

    let dest = DEST_TABLE.get(dest.as_str()).unwrap().to_string();
    let comp = comparing_for_comp(comp.as_str());
    let jump = JUMP_TABLE.get(jump.as_str()).unwrap().to_string();

    let into_binary = String::from("111") + &comp + &dest + &jump;
    return into_binary;
}

Both return the binary equivalent of each of their instructions.

The c_instruction funcion uses another function comparing_for_comp because it has two conditional. It dictated whether the a bit will be 0 or 1. I didn’t want to do all in one function so I left this job to another seperate function.

This is it. It’s very simple and easy. But my very limited experience with rust made me go back and forth between code and docs along with a lot of googling.

fn symbols(code_with_symbols: &mut Vec<&str>) -> Vec<String> {
    let mut symbol_table: HashMap<&str, usize> = HashMap::new();

    // --- snip ---

    let mut line_no: usize = 1;
    let mut label_index: Vec<usize> = Vec::new();

    // reading and removing lables
    loop {
        let mut line: &str = &code_with_symbols[line_no - 1];
        line_no += 1;

        if line.contains("(") {
            line = &line[1..line.len() - 1];
            let address: usize = line_no - 2;

            symbol_table.insert(line, address);
            label_index.push(line_no - 1);
            code_with_symbols.remove(line_no - 2); // line_no starts at 1 and is inc at beginning
        }

        if line_no == code_with_symbols.len() {
            break;
        }
    }

    // reading and inserting variables if new
    let mut no_of_new_variables: usize = 0;
    let mut line_no: usize = 0;
    let mut code_without_symbols: Vec<String> = Vec::new();
    loop {
        let mut line: &str = &code_with_symbols[line_no];
        line_no += 1;

        if line.contains("@") {
            // checking if just an address
            match line[1..].parse::<usize>() {
                Ok(_) => {
                    code_without_symbols.push(line.to_string());
                    continue;
                }
                _ => (),
            }
        }

        if line.contains("@") && (symbol_table.contains_key(&line[1..line.len()]) == false) {
            let address = 16 + no_of_new_variables;
            line = &line[1..];
            symbol_table.insert(line, address);
            let new_inst: String = format!("@{}", address);

            code_without_symbols.push(new_inst);

            no_of_new_variables += 1;
            continue;
        } else if line.contains("@") && (symbol_table.contains_key(&line[1..line.len()]) == true) {
            let address = symbol_table.get(&line[1..]).unwrap();
            let new_inst = format!("@{}", address);

            code_without_symbols.push(new_inst);
            continue;
        }
        code_without_symbols.push(line.to_string());
        if line_no == code_with_symbols.len() {
            break;
        }
    }

    return code_without_symbols;
}
